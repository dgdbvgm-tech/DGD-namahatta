# Миграция Банка Идей на Telegram Web App SDK

**Задача:** Переход от анонимной авторизации к жесткой привязке по `tg.initDataUnsafe.user.id` для исключения накруток голосов. Реализация заглушки-блокиратора при открытии вне Telegram.

## Контекст
В текущей версии MVP Банка Идей используется Firebase Anonymous Authentication. Это позволяет пользователям быстро и без препятствий участвовать в голосовании. Однако этот подход уязвим к накруткам (например, если пользователь очистит кэш или откроет сайт в режиме инкогнито, он получит новый анонимный UID и сможет голосовать заново).

## Требования к реализации
1. **Интеграция Telegram Web App SDK:** Подключить `https://telegram.org/js/telegram-web-app.js`.
2. **Проверка среды выполнения:** При входе в раздел "Банк Идей" или при загрузке приложения проверять, запущено ли оно внутри Telegram (`window.Telegram.WebApp.initData` не пусто).
3. **Заглушка вне Telegram:** Если пользователь открывает страницу через обычный браузер, блокировать доступ к функционалу голосования и показывать сообщение с кнопкой-ссылкой на Telegram бота/канал, через который нужно открывать Mini App.
4. **Обновление модели пользователей (Firestore):**
   - Перестать использовать `firebase.auth().signInAnonymously()`.
   - Использовать `tg.initDataUnsafe.user.id` как уникальный идентификатор пользователя (сохранять его в документе `Users/{tgUserId}`).
   - Для безопасности операций записи в Firestore желательно реализовать валидацию `initData` на backend-функции (Cloud Functions) или использовать кастомные токены Firebase Auth (Custom Auth с Telegram validation), так как прямая запись по ID клиента из `initData` без валидации может быть подделана злоумышленником.
5. **Миграция существующих голосов (опционально):** При необходимости придумать механику маппинга анонимных голосов на Telegram пользователей (или просто обнулить базу при релизе этой фичи).
